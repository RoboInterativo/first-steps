- `tail -f /var/log/nginx/error.log` - ошибки nginx

### **Неделя 4: Python-сайт и проксирование**

**Цель:** Научиться запускать Python-приложения, понять reverse proxy.

#### **ТЕОРИЯ:**

```
## Reverse Proxy

### Что это?
Nginx работает как "входные ворота". Пользователь стучится в nginx, а nginx перенаправляет запрос в приложение (Python, Node.js, и т.д.).

### Зачем?
- Одно приложение на разных доменах
- Балансировка нагрузки
- SSL-терминация
- Раздача статики (чтобы Python не занимался этим)

### Как это работает в конфиге:
location / {
    proxy_pass http://localhost:5000;  # куда перенаправлять
    proxy_set_header Host $host;        # передавать оригинальный Host
    proxy_set_header X-Real-IP $remote_addr;  # передавать реальный IP
}
```

**Задание 4.4: Конфиг nginx для прокси**
```bash
# 1. Создайте конфигурацию
cat > /etc/nginx/sites-available/timurpython.geekslore.ru << 'EOF'
server {
    listen 80;
    server_name timurpython.geekslore.ru;

    access_log /var/log/nginx/timurpython.access.log;
    error_log /var/log/nginx/timurpython.error.log;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Если добавите статические файлы (CSS, JS) - nginx будет их отдавать сам
    location /static {
        alias /var/www/timurpython.geekslore.ru/static;
    }
}
EOF

# 2. Включите сайт
ln -s /etc/nginx/sites-available/timurpython.geekslore.ru /etc/nginx/sites-enabled/

# 3. Проверьте и перезагрузите
nginx -t
systemctl reload nginx
```
